<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| adapter output + built shared payload -> target project filesystem | Lifecycle services must not write outside managed project paths or overwrite unmanaged edits silently. |
| installed runtime boundary -> arbitrary target project | Installed Researcher execution must not depend on host `tsx`, repo-local dependencies, or inherited module mode. |
| persisted install manifest + current file state -> inspect summary | Operators must be able to trust inspect output to explain what Researcher owns and what has drifted. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-03A | T | built runtime boundary | mitigate | Build self-contained runtime entrypoints and, where needed, managed package-boundary files so target projects do not need this repo's dependency graph. |
| T-06-04 | T | install and update services | mitigate | Restrict writes to managed relative paths, compare against persisted manifest records, and surface conflicts instead of forcing overwrites. |
| T-06-05 | I | inspect service | mitigate | Derive inspect summaries from the same manifest and managed-file checks used by install/update so operator output stays authoritative. |
</threat_model>
