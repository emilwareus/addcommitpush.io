# Track C: Practitioner and Vendor Guidance

## Cross-vendor Consensus

AI coding agents work best when the repo gives them persistent instructions, fast ways to find local context, and a hard verification loop.

## Repeated Guidance

- Put persistent repo guidance in version control.
- Keep it short, concrete, and scoped.
- Document build, lint, test, and validation commands.
- Make context discoverable through repo maps, code search, code graphs, MCP, and doc pointers.
- Break work into bounded tasks.
- Use plan-first workflows for ambiguous changes.
- Treat bad context as harmful.

## Strong Sources

- Anthropic Claude Code best practices and memory docs.
- OpenAI Introducing Codex, How OpenAI uses Codex, Docs MCP, internet access.
- GitHub Copilot repository and path-specific instructions.
- Cursor rules.
- Windsurf `AGENTS.md` and memories/rules.
- Sourcegraph Cody context.
- Aider repo map, conventions, lint/test docs.
- Augment guidelines and AGENTS.md guidance.
- Google Gemini CLI plan mode.

## Talk-ready Line

AI-ready codebases are not primarily about prompts. They are about operational context, scoped instructions, and verifiable workflows.
