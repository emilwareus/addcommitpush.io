# Track E: Latest 2025-2026 Direction

## Executive Summary

The frontier moved from "can an agent patch a bug?" to "can it find the right context, manage long horizons, preserve architecture, reuse memory, and know when not to edit?"

## What Is New Relative to 2023-2024

- Process supervision: context recall/precision, not just pass/fail.
- Long-horizon tasks: features, migrations, refactors, modernization.
- Memory and reuse: prior trajectories become measurable assets.
- Abstention and maintainability: correct behavior may mean no edit, and passing tests may still harm structure.
- Runtime architecture: compaction, sandboxing, async PR loops, approvals, and cloud environments are now core design elements.

## Talk-ready Frame

The future is not "prompt harder." The future is repository-scale context engineering: retrieve the right evidence, keep the working set stable, reuse past work, avoid unnecessary edits, and preserve architecture.
